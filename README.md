# 學生陳述紀錄單產生器

一個以單一 `HTML + CSS + JavaScript` 製作的單頁網頁應用，提供老師在校園現場快速記錄學生陳述內容、補充觀察、簽名確認、列印與本機保存使用。

## 專案特色

- 單一 `index.html` 即可直接開啟使用
- 手機優先設計，也支援電腦操作
- 支援學生陳述內容的語音轉文字輸入
- 支援學生與教師雙簽名畫布
- 使用 `localStorage` 保存草稿與歷史紀錄
- 支援歷史紀錄搜尋、載入編輯、刪除
- 支援單筆 JSON 匯出與匯入
- 支援 A4 直式列印，列印時自動隱藏操作介面

## 使用方式

1. 直接開啟 [index.html](./index.html)
2. 填寫基本資料、事件資訊與文字內容
3. 如瀏覽器支援，可在「學生陳述內容」使用語音輸入
4. 讓學生與教師於畫布區簽名
5. 點選「儲存紀錄」保存到本機瀏覽器
6. 需要備份時可匯出 JSON
7. 需要紙本時可直接列印

## 功能說明

### 1. 基本資料

- 紀錄編號會自動產生，格式為 `YYYYMMDD-001`
- 日期預設為當天
- 時間預設為目前時間

### 2. 事件資訊

- 事件性質可複選
- 陳述方式與情緒狀態可單選
- 若選擇「其他」，會顯示補充欄位

### 3. 學生陳述內容

- 可手動輸入
- 可透過瀏覽器原生語音辨識轉為文字
- 建議盡量保留學生原始說法，避免加入推測或定論

### 4. 教師補充與後續處理

- 可分別記錄背景摘要、教師補充與後續處理
- 內建常用語句快速插入按鈕

### 5. 確認與簽名

- 學生與教師皆可使用滑鼠或手指簽名
- 提供清空簽名功能
- 簽名會隨紀錄一起存入本機

### 6. 歷史紀錄

- 可依學生姓名或日期搜尋
- 可點選載入既有紀錄後繼續編輯
- 可刪除單筆紀錄

## 語音輸入注意事項

- 使用瀏覽器原生 `SpeechRecognition / webkitSpeechRecognition`
- 建議使用最新版 Chrome 或 Edge，以獲得較佳繁體中文辨識體驗
- 若瀏覽器不支援，系統會提示改用手動輸入
- 本工具不會保存錄音檔，只保留辨識後的文字內容

## 資料保存方式

本工具使用瀏覽器 `localStorage` 保存以下資料：

- 正式歷史紀錄
- 目前編輯中的草稿
- 每日紀錄編號流水號

請注意：

- 資料僅保存在目前使用的瀏覽器與裝置
- 清除瀏覽器網站資料後，紀錄可能消失
- 如需長期備份，建議定期匯出 JSON

## 列印說明

- 列印格式為 A4 直式
- 列印時會自動隱藏功能列與歷史紀錄區
- 只保留正式紀錄內容與簽名區

## 部署方式

此專案不需要後端。

可直接部署於：

- GitHub Pages
- Netlify
- Vercel 靜態網站
- 任意可提供靜態檔案的主機
